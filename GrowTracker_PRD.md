## 1. 產品概述

### 1.1 產品定位

> **「讓每一位家長都能輕鬆追蹤孩子的成長，數據背後是滿滿的愛。」**

「寶貝成長紀錄」是一款以 **煙火夜空** 為視覺主題的兒童成長追蹤 Web App，專為 0–10 歲嬰幼兒的家長設計。

核心訴求是讓非醫療專業的家長，能以最直覺的方式：

- 📏 記錄孩子的身高、體重
- 📊 即時對照 **WHO 生長標準百分位**
- 💊 追蹤保健品或配方奶的實際效果
- 📄 一鍵產出可分享的精美成長報告

---

## 2. 目標用戶

| 角色 | 描述 | 典型使用情境 |
|:----:|------|-------------|
| 👩 **主要用戶** | 有 0–10 歲孩子的台灣家長（中文為主） | 診所量測後，手機當場記錄 |
| 🌏 **次要用戶** | 海外台灣家長、英語系國家用戶 | 切換英文介面，分享給外籍配偶 |
| 👪 **共享用戶** | 被邀請查看孩子資料的家庭成員 | 祖父母想看孫子的成長曲線 |

## 3. 功能需求

| 功能 | 說明 | 狀態 |

| 登出 | 手動登出，清除本地 session | ✅ |

## 4. 非功能需求

### 4.1 多語系（i18n）

- 支援 **繁體中文**（預設）及 **English**
- 語言偏好存於 `localStorage`，切換後全站即時更新
- 所有 UI 文字、錯誤訊息、警示提示、月份名稱均隨語言切換
- 日期顯示使用自訂三欄式選單（年 / 月 / 日 `<select>`），不依賴系統原生 date picker，確保 iOS Safari 下語言顯示正確

---

### 4.2 Progressive Web App（PWA）

| 項目 | 說明 |
|------|------|
| `manifest.json` | App 名稱、圖示、主題色、顯示模式（standalone） |
| Service Worker | Cache-first 策略，離線可瀏覽已載入頁面 |
| 圖示 | `images/icon-512.png`：全色滿版方形，iOS 自行套用圓角遮罩 |
| iOS 主畫面 | 支援 `apple-touch-icon` 加到主畫面 |

---

### 4.3 無障礙（Accessibility）

| 項目 | 實作 |
|------|------|
| Modal | `role="dialog"` + `aria-modal="true"`，開啟時 focus trap，關閉後 focus 回觸發元素 |
| ESC 鍵 | 可關閉任何已開啟的 Modal |
| 折疊區塊 | `role="button"` + `tabindex="0"`，支援 Enter / Space 鍵觸發 |
| 日期選單 | 各欄位具備 `aria-label`（年 / 月 / 日） |
| Toast 通知 | 含 `aria-label` 關閉按鈕 |

---

### 4.4 資料安全

| 機制 | 說明 |
|------|------|
| **RLS（Row Level Security）** | Supabase 層級，用戶只能讀寫自己的資料 |
| **私有 Storage** | 孩子照片存於私有 Bucket，以 Signed URL（1 小時效期）存取 |
| **共享控制** | `child_shares` 資料表控制存取，Owner 可隨時撤銷 |

---

### 4.5 效能

- **html2canvas 懶加載** — 首次產報告時才載入，不影響首屏速度
- **WHO 資料本地化** — 以獨立 `data/who.js` 載入，無 CDN 依賴風險
- **Service Worker 預快取** — 所有靜態資源快取，重複訪問秒速載入

---

### 4.6 瀏覽器相容性

| 瀏覽器 | 支援程度 |
|--------|:-------:|
| Chrome（最新） | ✅ 完整 |
| Safari iOS / macOS | ✅ 完整（含自訂日期選單修正） |
| Firefox（最新） | ✅ 完整 |
| Samsung Internet | ✅ 完整 |
| Internet Explorer | ❌ 不支援 |

> 使用 ES2020 語法（`async/await`），不支援 IE。iOS Safari 的日期選單以自訂元件解決原生 date picker 無法跟隨 HTML `lang` 設定的限制。

## 5. 技術架構

### 5.1 前端架構

```
kids-growup-tracker-fireworks/
│
├── index.html          → HTML 結構（無模板引擎，純靜態）
├── style.css           → 全域樣式（CSS Variables 管理主題色）
├── app.js              → 全部前端邏輯
│   ├─ Toast / Modal 無障礙控制
│   ├─ 自訂日期選單（_dsReg 系統）
│   ├─ Auth 流程（Supabase onAuthStateChange）
│   ├─ 孩子 CRUD + 照片上傳
│   ├─ 量測 CRUD + 批次匯入
│   ├─ 成長速率計算（fmtAge、rateCalc）
│   ├─ 保健品 CRUD + 前後對比
│   ├─ Chart.js 圖表（身高 / 體重）
│   └─ 備份 / 還原（JSON）
├── i18n.js             → LANG 物件（zh / en），applyLang() 函式
├── db.js               → Supabase client + 所有 DB 操作封裝
├── report.js           → html2canvas 報告產生
└── data/who.js         → WHO 生長標準數據（0–120 個月）
```

---

### 5.2 後端（Supabase）

| 資料表 | 主要欄位 |
|--------|---------|
| `children` | `id`, `user_id`, `name`, `birthday`, `gender`, `father_height`, `mother_height`, `emoji`, `color_index`, `photo_url` |
| `measurements` | `id`, `child_id`, `date`, `height`, `weight`, `note`, `source` |
| `supplements` | `id`, `child_id`, `date`, `type`, `name`, `note` |
| `child_shares` | `id`, `child_id`, `owner_id`, `invitee_email` |

- 所有資料表均啟用 **RLS**
- Storage Bucket：`child-photos`（私有，Signed URL 存取）

## 6. 資料驗證規則

| 欄位 | 驗證規則 | 行為 |
|------|----------|:----:|
| 量測日期 | 不能是未來日期 | 🚫 阻擋 |
| 量測日期 | 不能早於孩子生日 | 🚫 阻擋 |
| 同日重複 | 已存在同日紀錄 | ⚠️ 提示確認 |
| 身高 | 有效範圍 40–200 cm | 🚫 阻擋 |
| 身高 | 比上次紀錄低 | ⚠️ 提示確認 |
| 身高 | 短期內增長幅度異常 | ⚠️ 提示確認 |
| 體重 | 有效範圍 2–100 kg | 🚫 阻擋 |
| 體重 | N 天內變動過大 | ⚠️ 提示確認 |
| Email | 共享邀請需符合 email 格式 | 🚫 阻擋 |
| Email | 不能邀請自己 | 🚫 阻擋 |
| Email | 不能重複邀請同一帳號 | 🚫 阻擋 |

> **設計哲學：** 可疑資料以「提示確認」而非「直接阻擋」，保留用戶自行決定的空間。

## 7. UX 設計原則

| # | 原則 | 說明 |
|---|------|------|
| 1 | 📱 **行動優先** | 主要使用情境是手機邊量邊記，UI 以手機為主設計，桌機作為延伸 |
| 2 | ⚡ **零等待感** | 登入後立即顯示首頁，資料在背景載入；不強制等待 API 回應才顯示 UI |
| 3 | 🔔 **操作即反饋** | 所有儲存 / 刪除操作均有 Toast 通知（成功 / 失敗） |
| 4 | 🛡️ **防呆不擋路** | 可疑資料（身高下降、重複日期）以提示確認，而非直接阻擋 |
| 5 | 🌐 **語言一致性** | 介面語言、月份名稱、錯誤提示、圖表 Tooltip 全部隨語言設定切換 |

## 8. 不在範圍內

以下功能**刻意排除**於目前版本：

| 功能 | 排除原因 |
|------|---------|
| 頭圍、BMI 等其他生長指標 | 目前僅聚焦身高 / 體重核心需求 |
| 推播通知（量測提醒） | 需要額外 Push 服務基礎建設 |
| 多帳號 / 組織管理 | 目前定位為個人 / 家庭工具 |
| 孩子資料細粒度權限（唯讀模式） | 目前共享者統一可查看並新增 |
| 原生 App（iOS / Android） | Web App 已滿足核心需求 |
| 離線寫入 | iOS 不支援 Background Sync API；Service Worker 目前僅快取讀取 |
