## 1. 產品概述

### 1.1 產品定位

「寶貝成長紀錄」是一款以煙火夜空為視覺主題的兒童成長追蹤 Web App，專為 0–10 歲嬰幼兒的家長設計。核心訴求是讓非醫療專業的家長，能以直覺的方式記錄孩子的身高體重、即時對照 WHO 生長標準、追蹤保健品或配方奶的效果，並一鍵產出可分享的成長報告。

### 1.2 目標用戶

| 角色 | 描述 |
|------|------|
| **主要用戶** | 有 0–10 歲孩子的台灣家長（中文為主） |
| **次要用戶** | 海外台灣家長、英語系國家用戶 |
| **共享用戶** | 被邀請查看孩子資料的家庭成員（如另一半、祖父母） |

### 1.3 核心價值主張

1. **資料長期保存** — 帳號登入、雲端同步，換裝置不怕資料消失
2. **醫學依據** — 對照 WHO P3–P97 百分位，一眼看出孩子在同齡中的位置
3. **效果追蹤** — 記錄保健品 / 配方奶，自動比較服用前後的年化成長速率
4. **分享友善** — 生成圖文報告，可列印、存圖、一鍵分享 LINE

## 2. 功能需求

### 2.1 帳號與認證

| 功能 | 說明 |
|------|------|
| Google OAuth 登入 | 使用 Google 帳號一鍵登入，無需另設密碼 |
| 自動登入 | 上次已登入的用戶，開啟 App 自動恢復 session |
| 登出 | 手動登出，清除本地 session |
| 帳號永久刪除 | 需輸入確認文字，刪除帳號下所有孩子、量測、保健紀錄，並登出 |

### 2.2 孩子管理

| 功能 | 說明 |
|------|------|
| 新增孩子 | 填入名字、生日（三欄式年月日選單）、性別（男 / 女），可選填父母身高 |
| 編輯孩子 | 修改名字、生日、性別、父母身高、表情符號、顏色主題 |
| 上傳照片 | 孩子頭像，儲存於 Supabase Storage 私有 Bucket，以 Signed URL 存取 |
| 多孩子 | 同帳號可管理多個孩子，首頁以卡片呈現選擇 |
| 刪除孩子 | 刪除單一孩子及其所有量測、保健紀錄 |
| 共享標記 | 被分享查看的孩子顯示「👥 共享」徽章 |

### 2.3 量測紀錄

| 功能 | 說明 |
|------|------|
| 新增量測 | 填入日期（三欄式選單）、身高（cm）、體重（kg）、備註（選填）並儲存 |
| 批次匯入 | 一次填入多筆歷史紀錄，每行含日期、身高、體重、資料來源；逐行驗證後整批寫入 |
| 歷史清單 | 依日期倒序列出，顯示日期、月齡、身高、體重、備註 |
| 編輯量測 | 修改已存紀錄的日期、身高、體重、備註 |
| 刪除量測 | 單筆刪除，附確認提示 |
| 資料驗證 | 未來日期、出生前日期 → 阻擋；同日重複紀錄、身高下降、異常跳動 → 提示確認 |

### 2.4 生長曲線圖（Charts）

| 功能 | 說明 |
|------|------|
| 身高曲線 | 以折線圖顯示孩子歷次身高，疊加 WHO P3/P10/P50/P90/P97 百分位帶 |
| 體重曲線 | 同上，對照體重百分位 |
| 年齡範圍 | 0–120 個月（0–10 歲）；可切換「自動範圍」/「全範圍」 |
| Tooltip | 懸停顯示日期、月齡、數值、所在百分位區間 |
| 資料來源標示 | 圖表下方顯示「WHO Child Growth Standards 0–10 歲」 |

### 2.5 成長速率分析

| 功能 | 說明 |
|------|------|
| 年化身高增長 | 以最新兩筆紀錄計算年化增長速率（cm/year） |
| 年化體重增長 | 同上，計算體重年化增長（kg/year） |
| 最低標準比較 | 依年齡顯示 WHO 最低建議速率，並計算距達標還差多少 |
| 遺傳身高預測 | 輸入父母身高後，以目前身高百分位推估預測成人身高（Mid-Parental Height 公式） |
| 醫療建議區間 | 依百分位自動顯示對應的警示文字及建議（P3 以下建議就醫） |

### 2.6 保健品追蹤

| 功能 | 說明 |
|------|------|
| 新增保健品 | 填入開始日期（三欄式選單）、類型（保健品 / 營養品 / 成長飲品 / 配方奶 / 其他）、品名、備註 |
| 編輯保健品 | 修改已存紀錄 |
| 刪除保健品 | 單筆刪除，附確認提示 |
| 前後對比 | 以各保健品開始日期為分界，計算並顯示服用前 / 後的年化身高增長速率 |
| 曲線標記 | 生長曲線圖上以綠色虛線標出各保健品開始日期 |

### 2.7 成長報告

| 功能 | 說明 |
|------|------|
| 報告內容 | 孩子基本資料、最新身高體重、百分位、成長速率分析、身高 / 體重曲線截圖 |
| 輸出格式 | 列印（`window.print()`）、存為圖片（html2canvas）、分享 LINE |
| 行動端按鈕 | 手機版合併為「儲存 / 分享」單一按鈕；桌機版分開為「列印」和「儲存圖片」 |

### 2.8 家庭共享

| 功能 | 說明 |
|------|------|
| 邀請家人 | 輸入家人 Gmail，對方用自己的 Google 帳號登入後可查看並新增此孩子的紀錄 |
| 移除共享 | 帳號擁有者可隨時移除特定家人的存取權 |
| 共享限制 | 不能共享給自己、不能重複邀請同一帳號 |

### 2.9 備份與還原

| 功能 | 說明 |
|------|------|
| 匯出 JSON | 將目前孩子的所有量測及保健紀錄匯出為 `.json` 備份檔 |
| 匯入 JSON | 從備份檔還原資料，重複日期自動略過 |
| 版本相容 | 偵測備份版本，較新版本提示確認後才匯入 |

## 3. 非功能需求

### 3.1 多語系（i18n）

- 支援繁體中文（預設）及 English
- 語言偏好存於 `localStorage`，切換後全站即時更新
- 所有 UI 文字、錯誤訊息、警示提示、月份名稱均隨語言切換
- 日期顯示使用自訂三欄式選單（年 / 月 / 日 `<select>`），不依賴系統原生 date picker，確保 iOS Safari 下語言顯示正確

### 3.2 Progressive Web App（PWA）

- `manifest.json` 定義 App 名稱、圖示、主題色、顯示模式（standalone）
- Service Worker（cache-first 策略）讓 App 可離線瀏覽已載入的頁面
- `images/icon-512.png` 作為 PWA 圖示及 iOS 主畫面圖示（apple-touch-icon）
- 圖示為全色滿版方形，iOS 自行套用圓角遮罩

### 3.3 無障礙（Accessibility）

- 所有 Modal 包含 `role="dialog"`、`aria-modal="true"`，開啟時 focus trap，關閉後 focus 回到觸發元素
- ESC 鍵可關閉任何已開啟的 Modal
- 折疊區塊（collapse header）具備 `role="button"` 及 `tabindex="0"`，支援 Enter / Space 鍵觸發
- 日期選單 `<select>` 各有 `aria-label`（年 / 月 / 日）
- Toast 通知含 `aria-label` 關閉按鈕

### 3.4 資料安全

- Supabase Row Level Security（RLS）：用戶只能讀寫自己的資料
- 孩子照片儲存於私有 Bucket，以 Signed URL（1 小時效期）存取
- 共享機制以 `child_shares` 資料表控制，Owner 可隨時撤銷

### 3.5 效能

- html2canvas 懶加載（首次產報告時才載入）
- WHO 資料以獨立 `data/who.js` 載入，無 CDN 依賴風險
- Service Worker 預快取所有靜態資源，重複訪問秒速載入

### 3.6 瀏覽器相容性

- 主要支援：Chrome、Safari（iOS / macOS）、Firefox、Samsung Internet
- 使用 ES2020 語法（`async/await`、`Array.prototype.slice.call`），無 IE 支援需求
- iOS Safari 的日期選單以自訂元件解決原生 date picker 無法跟隨 HTML `lang` 設定的限制

## 4. 技術架構

### 4.1 前端架構

```
index.html          → HTML 結構（無模板引擎，純靜態）
style.css           → 全域樣式（CSS Variables 管理主題色）
app.js              → 全部前端邏輯
  ├─ Toast / Modal 無障礙
  ├─ 自訂日期選單（_dsReg 系統）
  ├─ Auth 流程（Supabase onAuthStateChange）
  ├─ 孩子 CRUD + 照片上傳
  ├─ 量測 CRUD + 批次匯入
  ├─ 成長速率計算（fmtAge、rateCalc）
  ├─ 保健品 CRUD + 前後對比
  ├─ Chart.js 圖表（身高 / 體重）
  └─ 備份 / 還原（JSON）
i18n.js             → LANG 物件（zh / en），applyLang() 函式
db.js               → Supabase client + 所有 DB 操作封裝
report.js           → html2canvas 報告產生
data/who.js         → WHO 生長標準數據（0–120 個月）
```

### 4.2 後端（Supabase）

| 資料表 | 主要欄位 |
|--------|---------|
| `children` | id, user_id, name, birthday, gender, father_height, mother_height, emoji, color_index, photo_url |
| `measurements` | id, child_id, date, height, weight, note, source |
| `supplements` | id, child_id, date, type, name, note |
| `child_shares` | id, child_id, owner_id, invitee_email |

- 所有資料表均啟用 RLS
- Storage Bucket: `child-photos`（私有，Signed URL 存取）

### 4.3 部署與 CI/CD

- **部署**：GitHub Pages（`main` branch 直接服務）
- **CI**：GitHub Actions，push / PR 觸發 ESLint lint 檢查
- **Service Worker 版本**：`grow-tracker-v3`（每次靜態資源更新時遞增）

## 5. 資料驗證規則

| 欄位 | 規則 |
|------|------|
| 量測日期 | 不能是未來日期；不能早於孩子生日 |
| 同日重複 | 已存在同日紀錄 → 提示確認是否繼續 |
| 身高 | 40–200 cm（超出阻擋）；比上次低 → 提示確認 |
| 體重 | 2–100 kg（超出阻擋）；N 天內變動過大 → 提示確認 |
| 身高異常增長 | 短期內增長幅度超出預期 → 提示確認 |
| Email 格式 | 共享邀請需符合 email 格式；不能邀請自己；不能重複邀請 |

## 6. UX 設計原則

1. **行動優先** — 主要使用情境是手機邊量邊記，UI 以手機為主設計，桌機作為延伸
2. **零等待感** — 登入後立即顯示首頁，資料在背景載入；不強制等待 API 回應後才顯示 UI
3. **操作即反饋** — 所有儲存 / 刪除操作均有 Toast 通知（成功 / 失敗）
4. **防呆不擋路** — 可疑資料（身高下降、重複日期）以「提示確認」而非「直接阻擋」，保留用戶自行決定的空間
5. **語言一致性** — 介面語言、月份名稱、錯誤提示、圖表 Tooltip 全部隨語言設定一致切換

## 7. 不在範圍內（Out of Scope）

- 頭圍、BMI 等其他生長指標（目前僅身高 / 體重）
- 推播通知（量測提醒）
- 多帳號 / 組織管理
- 孩子資料的細粒度權限控制（目前共享者可查看並新增，無唯讀模式區分）
- 原生 App（iOS / Android）
- 離線寫入（目前 Service Worker 僅支援讀取快取）
